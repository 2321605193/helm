# AGENTS.md — Helm 项目治理规则

> 本文件始终加载。所有 Agent 必须遵守以下规则。

## 任务分级

| 级别 | 适用场景 | 门禁 | 模板 | 审批 |
|------|---------|------|------|------|
| **S** | Typo、注释、微小修改 | G4+G5 | 仅 verification.md | 自动 |
| **M** | Bug 修复、小功能 | G1-G5 | 全部模板 | 自动 |
| **L** | 跨模块、架构变更 | G0-G7 | 全部 + 回滚方案 | 确认后执行 |
| **CRITICAL** | 数据、权限、安全、发布 | G0-G8 + 人工 | 全部 + 安全评审 | 人工确认 |

不确定时默认按 **M** 级处理。

## 工具选择规则

- 读文件用 Read，不要用 cat/head/tail
- 搜索用 Grep，不要用 grep/rg
- 查找文件用 Glob，不要用 find/ls
- 编辑用 Edit，不要重写整个文件（除非文件很小）

## Git 协作规则

- 每个 task 完成后立即 git commit
- 使用原子提交，一个 commit 只做一件事
- Commit message 遵循 `type(scope): description` 格式
- 破坏性操作（force push、reset --hard、rm -rf）必须经用户确认

## 红线（不可违反）

1. **禁止编造测试结果** — 没跑的测试不能说通过了
2. **禁止硬编码密钥** — 使用环境变量或配置文件
3. **禁止静默吞错** — catch block 必须有日志或抛出
4. **禁止未确认的破坏性操作** — git push --force、DROP TABLE 等
5. **禁止把工具跳过包装成成功** — 工具失败必须如实报告

## 交付格式

每个任务完成后必须包含：
- 完成内容的说明
- 实际运行的验证命令和结果
- 未验证项或剩余风险
- 同步更新的文档列表
